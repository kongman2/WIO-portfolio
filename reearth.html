RE:EARTH

기부·환경 인증·세제 혜택·친환경 소비를 하나의 흐름으로 연결한 ESG 핀테크 플랫폼

"복잡한 절차 없이 쉽고 재미있게 참여하는 ESG 경험"

배포 링크

무료 배포 환경 특성상 초기 로딩 지연 및
이미지 업로드 오류가 발생할 수 있습니다.

핵심 MVP 전략

"환경 참여 → 보상 → 혜택 → 리포트로 이어지는 원스톱 ESG 여정"
MVP 포인트 의도
행동 인증하기 진입 장벽 완화
포인트 지급 즉각적 보상
기부/세제 혜택 연결 복잡한 제도 단순화
탄소 절감 리포트 성취 기반 재방문 유도
지도 기반 활동 확인 실시간 경험 제공

→ 단순 정보 제공이 아니라 참여가 누적되고 보이는 구조에 집중했습니다.

사용자 플로우

인증 → 포인트 → 기부·혜택 → 소비·커뮤니티 → 리포트

주요 기능

  • 환경 행동 인증 & 포인트 지급 (공공 자전거/재활용/헌옷 기부)
  • 기부 + 세제 혜택 자동 처리
  • 친환경 포인트 샵
  • 커뮤니티, 랭킹, 챌린지
  • ESG 활동 리포트 & 대시보드
  • 지도 기반 사용자 활동 위치 확인 (카카오맵 API)

UX 설계 & 디자인 아이덴티티

"지속 가능한 ESG는 재미와 보상이 있는 구조에서 시작한다."

브랜드 감성 + UX 구조를 결합한 경험 설계

요소 설계 목적
GSAP 랜딩 인터랙션 메시지 톤을 시각적으로 각인
컬러 시스템·아이콘 ESG 브랜드 성향 반영
리포트 UI 시각화 숫자를 '경험'으로 전환
카드 기반 UI 패턴 확장성과 일관성 확보
지도 기반 인증 UI 참여 결과를 감각으로 전달
반응형 레이아웃 (Bootstrap 활용) 디바이스별 동일 경험

그래픽 요소를 서비스의 태도와 사용자 행동을 안내하는 언어로 사용했습니다.

결과 & 배운 점

  • ESG 브랜드를 기반으로 UI·그래픽 시스템 구축
  • GSAP 인터랙션을 활용하여 브랜드 메시지의 시각적 인지와 행동 유도를 동시에 달성
  • 배지·포인트·챌린지를 기반으로 한 게이미피케이션 구조를 설계하여 지속 참여 방향성 제시
  • 복잡한 ESG 활동을 하나의 사용자 흐름으로 정리해 접근성 개선

첫 번째 프로젝트에서는 기획·디자인에 과도하게 시간을 투자하면서 개발 기간이 부족해졌고, 협업 과정에서는 역할 정의가 명확하지 않아 중복 구현이 발생하는 문제를 겪었습니다. 이 경험을 바탕으로 이후 프로젝트에서는 오토레이아웃, 컴포넌트 기준, 스타일 가이드, 명명 규칙 등을 초기에 정리하여 UI의 일관성 확보와 개발 효율 향상을 우선 순위로 두었습니다. 또한 인터랙션을 단순히 시각적으로 '예쁘게 보이기 위한 요소'가 아니라 브랜드의 태도와 서비스 경험의 흐름을 전달하는 중요한 설계 요소라는 관점을 갖게 되었습니다.
