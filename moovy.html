MOOVY

넷플릭스 트렌드를 기반으로 대화와 반응이 빠르게 축적되는 참여형 영화 리뷰 플랫폼

"기록 저장형 리뷰가 아닌, 트렌드 중심 커뮤니티 경험"

정식 배포 미진행, 내부 테스트 환경
무료 배포 환경과 데이터 API 특성상 업데이트 주기는 제한적

핵심 MVP 전략

"참여-보상-혜택-피드백"이 돌아오는 ESG 루프 설계
MVP 기능 목적
사진 인증 기반 참여 진입 장벽 최소화
즉각 포인트 보상 행동 강화
기부/세제 혜택 연결 실질적 가치 제공
탄소 절감 리포트 성취감 & 지속성
지도 기반 활동 피드백 참여의 실감형 경험

→ 단순 참여가 아닌, '돌아오고 싶게 만드는 구조'가 핵심

사용자 플로우

메인 → 영화 리스트/검색 → 영화 상세 → 리뷰/답글 → 마이페이지(캘린더)

주요 기능

  • 넷플릭스 TOP 랭킹 데이터 자동 수집
  • 신작/현재 상영작 TOP3 자동 노출
  • 관리자 추천작 큐레이션 탭
  • 기간제 토픽 + 댓글/답글/좋아요
  • 마이페이지 기록 히스토리(캘린더 저장)
  • IntersectionObserver 기반 무한 스크롤

디자인 시스템 & UI 설계

"MOOVY는 패턴 중심 UI 시스템 설계가 핵심이었습니다."
설계 요소 의도
컴포넌트 기반 화면 패턴 정의 구조적 UI 개발 협업
카드/리스트/모달 공통화 유지보수 & 디자인 일관성
타입 기반 시스템(기획 관점) 상태/선택/필터의 예측 가능성
검색/필터/페이지네이션 방대한 데이터 탐색 UX 개선
반응형 레이아웃 (Bootstrap 활용) 디바이스별 동일 경험

이 프로젝트에서 "스타일 시스템 → 화면 설계 → 개발" 순서를 명확히 적용했습니다. PETHAUL/RE:EARTH에서 느낀 시행착오를 기반으로 성장한 부분이라고 생각합니다.

결과 & 배운 점

  • 분산된 영화 정보 탐색을 하나의 상세 경험으로 통합
  • 트렌드 기반 콘텐츠로 참여 전환률 향상 방향성 확인
  • 공통 UI 컴포넌트 설계로 디자인·개발 일관성 확보
  • MVP 범위에서 반응이 쌓이는 커뮤니티 구조 검증

Figma의 Variant 구성 기준과 Design Token(Variable) 정의 부분에서는 실무적으로 요구되는 수준에 미치지 못한 점을 인지했고, 현재는 이를 보완하기 위해 컴포넌트 단위 설계 · 변수화 전략 · 라이브러리화 시나리오를 지속적으로 학습하고 있습니다.
